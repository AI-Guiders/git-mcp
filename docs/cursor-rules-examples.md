# Cursor: примеры правил (копипаст)

Сохрани как `.cursor/rules/git-logical-commits.mdc` (или своё имя).

~~~mdc
---
description: Git — разбивать изменения на логические коммиты, не один большой коммит
alwaysApply: true
---

# Логические коммиты

При коммите и пушЕ **разбивать изменения на несколько коммитов по смыслу**, а не делать один общий коммит.

Примеры разбивки:
- отдельный коммит: новая фича (парсер, экран, сущность + миграция);
- отдельный коммит: рефакторинг/переименование;
- отдельный коммит: правки UI (подвкладки, подписи);
- отдельный коммит: конфиг/доки/ресурсы, если они логически отдельны.

Так история остаётся читаемой и откат/cherry-pick по коммитам проще. **Переписывать уже запушенную историю (rebase/force-push) не делать** — только новые коммиты разбивать логически.
~~~
